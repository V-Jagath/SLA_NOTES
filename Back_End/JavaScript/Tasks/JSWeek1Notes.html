DAY 1 TASKS

1. Implicit Conversion & Explicit Conversion

  • Implicit Conversion: JavaScript automatically converts one data type to another without you writing code. Ex: console.log("5" == 5)
  • Explicit Conversion: You convert type manually using functions. Ex: console.log(String(5)) → convert to string

2. Difference between return and console.log

  • return: Sends output from a function to the place where the function was called. Ends the function immediately.
  • console.log: Prints message to the browser console. Used only for debugging.

3. Lexical scope and Closures

  • Lexical scope: A function can access variables from its parent (outer) scope. Outer function variables are available inside inner functions.
  • Closures: Closure = Function + its outer scope variables stored together even after the outer function is finished. Used to create private data.

4. Differences between null and undefined

  • Null: A value that represents nothing, empty, or no data — but intentionally set.
  • Undefined: A variable is declared but not given a value.

5. Array, Object, Array of Object and JSON

  • Array: An ordered list of values stored in index format. Access → console.log(fruits[1]);
  • Array Of Object: An array where each element is an object. Access → console.log(students[1].name);
  • JSON: JavaScript Object Notation. A data format used to send/receive data between backend and frontend. Access → fetch method

Functions

Function Declaration

  • Uses function keyword.
  • Fully hoisted (can call before definition).

  • Not hoisted.

  • Uses =>.

// 3. Remaining Object Props
const {name, ...others} = {name:"Jagan", age:22, city:"Chennai"};

Tasks: What is a Callback?

A callback is a function passed as an argument to another function, so that the other function can call it later.

Uses

  • To handle asynchronous operations (Fetching data, Timers, Database queries).
  • To control the order of execution (avoid blocking the program).
  • To reuse logic (flexibility).
function greet(name, callback) {
    console.log("Hello " + name);
    callback(); // calling the callback function
}
function sayBye() {
    console.log("Bye!");
}
greet("Jagath", sayBye);

Day - 3 Loops and Closures (26.11.2025)

Expression

An expression is any code that produces a value. Example: var a = 10;

If Statement

If uses logical (&&, ||, !) or comparison operators (==, ===, >, <, >=, <=).

Why return inside if? return is used to stop the function and send a value back. Useful when a condition must end the function early.

For Loop (Very Important for Interviews)

Most common loop in coding tests, used for arrays, strings, objects, and logic patterns.

for (let i = 0; i < 5; i++) {
    console.log(i);
}

Let (Interview Fact)

Why “let” is preferred? Block-scope → safer. Prevents accidental re-declaration. Best for functional and modern JavaScript.

Function Call Types

  1. Open Call: You call the function directly: show();
  2. Event Call: Function called through an event: button.onclick = show;

A function can return any value: function add(a,b){ return a+b; }

Lexical Scope

Definition: Inner function can access outer function variables. Outer function cannot access inner function variables.

const outer = () => {
    let a = 10;
    const add = () => {
        console.log(a); // inner can access outer
    }
    add();
}

Closures

Definition: A closure is when a function remembers its outer scope variables even after the outer function has finished executing.

Why we use closures? To create private data and to protect variables from outside access.

const createAcc = () => {
    let balance = 1000; // private variable
    const deposit = (amount) => {
        balance += amount;
        console.log("Balance:", balance);
    };
    // ... withdraw function ...
    return { deposit, withdraw };
};

Important Closure Rule: Inner function can access outer variables. Outer function CANNOT access inner variables.

Day-4 Callback and Array Methods (27.11.2025)

Callback Function

A callback function is a function that is passed as an argument to another function and gets executed after a task is completed.

Why do we use callbacks? To handle Asynchronous operations, API calls, Time-based tasks (setTimeout, setInterval), and tasks that take unknown time.

Promises

A Promise is an object that represents the eventual completion of an asynchronous task. It helps convert callback-based async code to cleaner async code and avoids callback hell. Promises have 3 states: Pending → Resolved → Rejected.

JavaScript Synchronous or Asynchronous?

JavaScript is **Synchronous** by default and **Single-threaded**, but supports **Asynchronous** behavior using Callbacks, Promises, async/await, and the Event loop.

Types of Callback Functions

  1. Synchronous Callback: Runs immediately during the execution of the main function.
  2. Asynchronous Callback: Runs later, after some time or after an async task is completed (Used in API calls, Timers, Fetch).
  3. Anonymous Callback: A callback function without a name (Used most commonly in array methods).
  4. Named Callback: A callback function with a name, passed as an argument.

Array Methods — Expanded Interview Answers

  • push(): Adds element(s) to the end. **Modifies original array.** Returns new length.
  • pop(): Removes the last element. **Modifies original array.** Returns the removed element.
  • unshift(): Adds element(s) to the beginning. **Modifies original array.** Returns new length.
  • shift(): Removes the first element. **Modifies original array.** Returns the removed value.
  • slice(start, end): Returns a portion of the array. **Does NOT modify original array.**
  • splice(start, deleteCount, items...): Modifies the original array (remove, add, replace). **Modifies original array.** Returns the removed elements.
  • concat(): Combines two or more arrays. **Does NOT modify original arrays.** Returns a new array.
  • indexOf(value): Returns the first index of a value, or -1 if not found.
  • includes(value): Checks if a value exists, returns true or false.
  • reverse(): Reverses the array in place. **Modifies original array.**
  • join(): Converts array elements into a single string with a separator.

Naming Conventions (Short Notes)

  • camelCase: firstName (JS variables/functions)
  • PascalCase: UserProfile (Classes, React components)
  • snake_case: user_name (Python, SQL, file names)
  • kebab-case: text-center (CSS/Tailwind/Bootstrap, not allowed in JS variables)

Higher Order Function (HOF)

A function that takes another function as an argument, returns a function, or does both. Used in: map, filter, reduce, callbacks.

Higher-Order Array Methods (Require Callback & Work Only on Arrays)

  • forEach(callback): Executes a callback for every element. Used only for iteration/side effects. **Does NOT return a new array.**
  • map(callback): Loops through each element and returns a **new array** with the transformed values.
  • filter(callback): Returns a **new array** containing only the elements that pass the condition (true).
  • find(callback): Returns the **first matching element**. If no match → undefined.
  • findIndex(callback): Returns the **index** of the first matching element. If no match → -1.
  • reduce(callback, initialValue): Reduces the array to a **single value** (sum, total, object, etc.).
  • some(callback): Returns true if **any one** element matches the condition.
  • every(callback): Returns true only if **all** elements pass the condition.
  • sort(): Sorts the array **in place**. For numbers, use a compare function: (a, b) => a - b.

Shallow Copy vs Deep Copy (Very Important for Interviews)

  • Shallow Copy: Creates a new array, but inner objects/arrays still refer to the original memory. Changing nested values affects the original. Methods: slice(), concat(), map(), filter(), spread operator [...].
  • Deep Copy: Creates a completely independent copy, including all nested objects. Changing deep values does NOT affect the original. Techniques: JSON.parse(JSON.stringify(array)), structuredClone(array).

Expression vs Statement (Very Important for Interviews)

  • Statement: Performs an action. Does not return a value. (e.g., if, for, let a;)
  • Expression: Produces a value. Can be used inside statements. (e.g., 10 + 20, x > 5)

Day - 5 ES6 + Features (28.11.2025)

1. LET and CONST

ES6 introduced let and const to fix issues with var (function-scoped, allows re-declaration, hoisted and initialized as undefined).

  • LET: Block scoped, can be updated, cannot be re-declared, hoisted but not initialized (TDZ error).
  • CONST: Block scoped, cannot be updated, cannot be re-declared, **must be initialized** at the time of declaration.

2. Arrow Function — Key Features

  • Short and Clean Syntax: const add = (a, b) => a + b;
  • No function keyword & No return needed (for one-line).
  • No Own this: Uses parent scope’s this.
  • Not usable as Constructor (new not allowed).
  • No arguments object (must use rest parameter).
  • Always Anonymous (unless assigned to a variable).

3. Template Literals (``)

Used for multi-line strings and ${ } for value insertion.

4. Destructuring (ES6 Feature)

Taking values from objects/arrays and storing them in separate variables easily.

  • Object Destructuring: Based on **KEY**. const { name, year } = obj;
  • Array Destructuring: Based on **POSITION**. const [first, ...second] = arr;

5. Default Parameter (ES6 Feature)

If you don’t pass a value, JavaScript will use the default value. const app = (name = "React") => { ... }

6. Import and Export (ES6 Modules)

Feature Default Export Named Export
How many? Only 1 per file Many
Import syntax import sub from "./path"; (no { }) import { add } from "./path"; (must use { })
Rename? Yes Yes (with as)

7. Optional Chaining (?)

Used to safely access nested object values without getting an error. If the left side is null or undefined, it stops and returns undefined instead of an error. Example: users?.name?.field

8. Nullish Coalescing Operator (??)

If the left side is null OR undefined, then show the right side value. It does NOT use the right side if the left side is 0, false, or "". 

let a = null;
console.log(a ?? "I am Showing"); // Output: I am Showing

9. Nullish Logical Assignment (??=)

Assign the right-side value only if the left-side value is null or undefined. (ES2021/ES12) 

let a;
a ??= 100; // a becomes 100

10. FOR…OF (Used for Arrays)

Used to loop through array **values**. Gives value directly. **Does NOT work for objects.**

11. FOR…IN (Used for Objects)

Used to loop through object **keys** (property names).

Array Methods Quick Review

  • map(): Creates a new array, used to modify values.
  • filter(): Creates a new array, used to select items based on a condition.
  • find(): Returns the first matching value (single value).
  • reduce(): Reduces array to one output (sum, max, etc.).
  • includes(): Checks if a value exists, returns true/false.

Week-1 Test Questions (29.11.2025)

JavaScript Interview Questions & Answers

1. What is the difference between var, let, and const in JavaScript?

  • var: Function-scoped, hoisted (undefined), allows redeclaration and reassignment. Not recommended.
  • let: Block-scoped, hoisted (TDZ), allows reassignment, **no** redeclaration in the same scope. Safer.
  • const: Block-scoped, hoisted (TDZ), **must** be initialized, **no** reassignment, **no** redeclaration. Content of objects/arrays can change, but reference cannot.

2. Why is the Temporal Dead Zone (TDZ) important?

TDZ exists from the start of the scope until the variable is initialized. It prevents using variables before they are initialized, helping catch logical errors early and making ES6 variables safer and predictable.

console.log(a); // ReferenceError
let a = 10;

3. Difference between =, ==, ===

  • = (Assignment Operator): Assigns a value.
  • == (Loose Equality): Compares only values, performs type conversion. (e.g., 7 == "7" is true)
  • === (Strict Equality): Compares value **and** datatype, no type conversion. (e.g., 7 === "7" is false)

4. Purpose of Spread Operator (…) and Rest Operator (…)

  • Spread Operator: Used to expand arrays/objects (e.g., copy, merge, add elements). let b = [...a];
  • Rest Operator: Used to collect parameters into an array. function add(...nums){ return nums; }

5. What are arrow functions? How are they different?

Shorter function syntax from ES6. Differences from normal functions: No this binding, no arguments object, cannot be used as constructors, automatically returns if single-line.

6. What is destructuring?

Destructuring extracts values from arrays/objects easily.

  • Object Destructuring: Based on **KEY**. const {name, age} = obj;
  • Array Destructuring: Based on **POSITION**. const [course1, course2] = arr;

7. How to access values in deeply nested objects/arrays?

Use dot/bracket notation or the modern **Optional Chaining** (obj?.user?.details?.name) to avoid ReferenceError.

8. Ways to clone an array or object

  • Array Clone: Spread ([...arr]), Slice (arr.slice()), Deep clone (JSON.parse(JSON.stringify(arr))).
  • Object Clone: Spread ({...obj}), Object.assign({}, obj), Deep clone (JSON.parse(JSON.stringify(obj))).

9. Difference between for…in, for…of, forEach(), and map()

  • for…in: Used for objects, loops **keys**.
  • for…of: Used for arrays, loops **values**.
  • forEach(): Array method, loops each element, **no return value**.
  • map(): Array method, returns a **new transformed array**.

10. What is a closure?

A function that remembers variables from its outer function even after the outer function is done. Used for counters, private variables, etc.

function outer(){
    let count = 0;
    return function(){
        count++;
        return count;
    };
}

11. What is a callback function and why is it needed?

A function passed as an argument into another function. Used in asynchronous programming, API fetching, timers, and event handling.

12. Difference between Function Declaration, Function Expression, and Arrow Function

  • Declaration: Hoisted. function greet(){ }
  • Expression: Not hoisted. const greet = function(){ };
  • Arrow: Short syntax, no this. const greet = () => "Hello";

13. What are higher-order functions?

A function that accepts another function **OR** returns another function **OR** both. Examples: map(), filter(), setTimeout().
